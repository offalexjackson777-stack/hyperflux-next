# Support

HyperFlux Next is unreleased. There is currently no supported installation or
package channel, but public source, generated documentation, and development
discussion are available.

## Choose The Right Route

| Need | Next action |
| --- | --- |
| Understand the project | Read the [project overview](docs/user/overview.md) |
| Check installation availability | Read [installation status](docs/generated/installation.md) |
| Ask a usage or development question | Open a [Help discussion](https://github.com/offalexjackson777-stack/hyperflux-next/discussions/categories/help) |
| Explore an early idea | Start an [Ideas discussion](https://github.com/offalexjackson777-stack/hyperflux-next/discussions/categories/ideas) |
| Report incorrect behavior | Use the [bug form](https://github.com/offalexjackson777-stack/hyperflux-next/issues/new?template=bug_report.yml) |
| Add public device research | Use the [hardware research form](https://github.com/offalexjackson777-stack/hyperflux-next/issues/new?template=hardware_research.yml) |
| Propose exact physical evidence | Use the [device qualification form](https://github.com/offalexjackson777-stack/hyperflux-next/issues/new?template=device_qualification.yml) |
| Report a documentation problem | Use the [documentation form](https://github.com/offalexjackson777-stack/hyperflux-next/issues/new?template=documentation.yml) |
| Report a vulnerability | Follow [SECURITY.md](SECURITY.md) and report privately |

## Privacy-Safe Diagnostics

When a future installed candidate provides `hyperfluxctl`, run its Doctor and
support-preview commands before opening a bug. A preview must not upload data or
query hardware. Share only the bounded reference and reviewed preview requested
by the issue form.

Never publish serials, stable host identifiers, private paths, raw transport
payloads, captures, memory dumps, credentials, or arbitrary logs. Remove any
such material before submitting a public report.

## Boundaries

Public support channels do not authorize hardware writes, active receiver
queries, suspend tests, pairing changes, package publication, or a product
release. Maintainers will move a report to private handling when security or
privacy requires it, and may close requests that cannot stay within these
boundaries.
