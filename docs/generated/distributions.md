## Lifecycle Contract

- Fresh installation creates conservative read-only configuration and does not enable the bridge.
- Reinstall and compatible update preserve configuration and resume only a previously active bridge.
- A changed in-use kernel module keeps the bridge stopped and reports reboot or explicit receiver-disconnect activation.
- Start preparation validates the installed module and migrates configuration before the bridge.
- Start confirmation removes rollback state only after the bridge has started successfully.
- Uninstall stops the bridge but leaves package-manager-managed configuration policy explicit.

## Python Integration

Pure Python modules live at `/usr/lib/hyperflux-next/python`. Each distribution adds one native `.pth` file so application entry-point discovery works without binding the common payload to a Python minor version.
