# Local Companion Contract

> Generated by `./hfx generate`. Do not edit manually.

The local companion gives the installed qualification console a privacy-safe,
versioned view of an installed HyperFlux system. The website itself cannot detect
a driver, inspect USB, or write hardware.

## Read Boundary

- Endpoint: `GET /v1/snapshot` on `http://127.0.0.1:47427`.
- Transport: loopback-only HTTP with bounded JSON.
- Default access: read-only.
- Device evidence states: `Active`, `Sleeping`, `Disconnected`, `Unknown`.
- Pairing and a retained receiver route never imply that a device is active.

## Evidence States

| State | Meaning |
| --- | --- |
| `Active` | Fresh receiver evidence shows the device is currently active. |
| `Sleeping` | The paired device is retained, but bounded evidence identifies sleep. |
| `Disconnected` | The current generation has explicit route-removal evidence. |
| `Unknown` | Evidence is absent, stale, or insufficient for a stronger statement. |

## Optional Writes

Writes are disabled by default. A future lighting write requires every condition:

- `explicit-user-confirmation`
- `bounded-scope`
- `expiry`
- `single-local-origin`

The maximum capability lifetime is 300 seconds.
Closing or expiring the capability returns the companion to read-only behavior.

## Privacy And Transport

Hardware serials and stable host identifiers are excluded. The companion never
uploads evidence silently, never uses WebHID, and never opens USB directly. All
hardware authority remains in the installed HyperFlux bridge and kernel path.
