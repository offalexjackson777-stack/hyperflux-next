## Clock And Ordering Semantics

`MonotonicMs` values belong to the serving bridge process monotonic clock. They are comparable only while `server_instance_id` remains unchanged; they are never wall-clock timestamps and must not be retained across bridge restart.

Lease expiries and transaction deadlines are exclusive absolute instants in that clock domain. Transaction `frame_index` values are contiguous, zero-based, and ordered within one atomic request. Terminal outcomes are immutable; a client looks them up instead of replaying a request after uncertain transport.
